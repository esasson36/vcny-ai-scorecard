# VCNY AI Scorecard

A monthly internal tool for VCNY Home staff to self-report their AI tool usage. Employees submit a scorecard rating how frequently they use tools like ChatGPT, Claude, and Perplexity, how much time it saves them, and its impact on their work. Admins can view, score, and export the results.

**Live site:** https://vcny-ai-scorecard.onrender.com

---

## Features

- **Employee submit form** — rate AI tool usage across 5 dimensions (frequency, time saved, impact, adoption, output volume)
- **Admin dashboard** — view all submissions, filter by month, see scores and grades per person
- **Leaderboard & team views** — ranked scores and adoption rates by team
- **CSV export** — download all submissions as a UTF-8 Excel-compatible spreadsheet
- **Print view** — print a clean one-page scorecard for any submission
- **Duplicate check** — prevents the same person from submitting twice in a month

## Running Locally

### 1. Prerequisites
- Node.js 18+
- A Supabase project with the tables created (see below)

### 2. Create Supabase tables
In your Supabase project → SQL Editor → New query, run:

```sql
create table if not exists submissions (
  id         text primary key,
  name       text not null,
  team       text not null,
  tools      text not null,
  use_cases  text default '',
  challenges text default '',
  timestamp  text not null,
  month      text not null default ''
);

create table if not exists headcounts (
  team  text primary key,
  count integer not null default 0
);
```

### 3. Set up environment variables
```bash
cp .env.example .env
```
Fill in `SUPABASE_URL` and `SUPABASE_SERVICE_KEY` from your Supabase project's **Settings → API** page.

### 4. Install and run
```bash
npm install
npm run dev
```

The app runs at `http://localhost:5000`.

## Deployment (Render)

The project is configured for Render via `render.yaml`.

**Required environment variables in Render dashboard:**
| Key | Where to find it |
|---|---|
| `SUPABASE_URL` | Supabase → Project Settings → API → Project URL |
| `SUPABASE_SERVICE_KEY` | Supabase → Project Settings → API → Secret key |
| `SESSION_SECRET` | Auto-generated by Render |

Render auto-deploys on every push to `master`.

## Scoring

Each submission scores up to 5 points across 4–5 dimensions:

| Dimension | Max |
|---|---|
| Frequency | 5 |
| Time Saved | 5 |
| Impact | 5 |
| Adoption | 5 |
| Output Volume *(optional, admin-entered)* | 5 |

**Grades:**
| Score | Grade |
|---|---|
| 80–100% | A |
| 60–79% | B |
| 40–59% | C |
| 20–39% | D |
| 0–19% | F |
